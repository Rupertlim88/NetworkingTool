  2. **Shadow approach settled**: Black 4-layer shadows (Aurora structure) on all themes. Card-colored and bright-teal shadows were invisible on dark backgrounds. Accepted that dark UIs can't replicate Aurora's shadow depth ŌĆö the brighter card surface + border + gradient + edge highlight provides depth instead.

  3. **Quadratic grid scaling implemented** (`lib/grid-scaling.ts`):
     - Gap grows proportionally to card area (width┬▓), not card width
     - Solves: N*w + (N-1)*k*w┬▓ = available_space at each breakpoint
     - k calibrated at each threshold to maintain 0.42 gap:card ratio
     - All 3 breakpoints (2/3/4 col) use uniform 0.42 ratio
     - JS resize handler sets --card-width, --card-gap, --row-gap CSS vars
     - CSS uses var() with fallbacks for pre-JS render
     - 1-column case left as linear (no column gap to scale)

  4. **Shadow scaling implemented**: All 4 shadow layers scale dynamically via JS.
     - Scales based on row gap at 40% rate: `s = 1 + (rowGapDelta * 0.4) / 30`
     - Rationale: shadow should bridge ~40% of row gap to connect rows without overwhelming cards
     - Previous attempts: scaled with gap (too aggressive), scaled with card width (shadow didn't bridge rows)

  5. **Border color exploration** (IN PROGRESS):

<blockers>
- **GSAP stacking context constraint**: Pseudo-elements cannot layer behind GSAP-transformed elements. Must use box-shadow or sibling DOM elements. Permanent constraint.
- **Dark-on-dark shadow limitation**: Black shadows are invisible on dark backgrounds regardless of approach. Accepted ŌĆö using surface brightness + border + gradient for depth instead.
- **Box-shadow layer ordering**: Multiple shadows on same element render first-on-top. A light glow as the last layer gets buried under dark layers above it. Use separate elements for independent shadow rendering.
</blockers>
