**Implementation spec for border effects (to implement later):**

The user wants three combined effects on each card:

1. **Static border**: 3px solid, using theme's secondary accent color (cyan #22d3ee for teal, coral #ff6b6b for midnight, pink #ec4899 for aurora). Opacity varies by star level: 1-star=30%, 2-star=60%, 3-star=100%. Must use inline styles for border (CSS variables don't work in border shorthand).

2. **Pulsing glow**: CSS @keyframes animating box-shadow. Three separate keyframes per star level with different intensities and cycle speeds:
   - 1-star: faint glow, 5s cycle
   - 2-star: medium glow, 4s cycle
   - 3-star: strong glow, 3s cycle
   Must NOT set box-shadow via inline styles — CSS animations cannot override inline styles. Use data-stars attribute + CSS selectors.

3. **Hover surge**: On hover, stop pulse (animation: none), snap glow to full brightness, scale 1.02x.

Key implementation notes:
- CSS variables in `border` shorthand don't work — use inline rgba values or split border-width/style/color
- box-shadow must be controlled entirely via CSS (not inline) for animations to work
- `overflow-hidden` on the card clips ::after pseudo-elements — avoid that approach
- `color-mix()` with CSS variables was unreliable — use direct rgba values
- Use `data-stars` attribute on card div for CSS targeting
- The card class was `contact-card-inner` during prototyping — can use any class name
</decisions_made>
