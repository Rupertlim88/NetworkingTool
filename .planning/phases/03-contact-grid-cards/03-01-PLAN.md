<task type="auto">
  <name>Task 2: Build grid page with responsive layout, gradient bleed, hidden scrollbar, and GSAP entrance animation</name>
  <files>app/page.tsx, app/globals.css</files>
  <action>
**1. Update `app/globals.css`:**
- Add `.contact-card` hover styles: `transform: scale(1.02)`, box-shadow combining `var(--shadow-card)` and `var(--shadow-card-hover-glow)`
- Add `.contact-card` base transition: `transition: transform 200ms ease, box-shadow 200ms ease`
- Add hidden scrollbar styles to `body`: `scrollbar-width: none` (Firefox), `-ms-overflow-style: none` (IE/Edge), and `::-webkit-scrollbar { display: none }` (Chrome/Safari)
- Add `.contact-card` initial state for animation: `opacity: 0` (prevents flash before GSAP runs)

**2. Replace `app/page.tsx` entirely:**
- `'use client'` directive (required for GSAP DOM access)
- Import: `gsap` from `'gsap'`, `ScrollTrigger` from `'gsap/ScrollTrigger'`, `useGSAP` from `'@gsap/react'`, `useRef` from `'react'`
- Import `ContactCard` from `@/components/ContactCard`, `MOCK_CONTACTS` from `@/lib/mock-data`
- Register `gsap.registerPlugin(ScrollTrigger)` at module scope (NOT inside component)
- Component structure:
  ```
  <div style={{ background: 'var(--color-grid-bg)', minHeight: '100vh' }}>
    <!-- Fixed gradient bleed overlay -->
    <div className="fixed inset-x-0 top-0 pointer-events-none z-0"
         style={{ height: '20vh', background: 'var(--gradient-grid-bleed)' }} />

    <!-- Grid container with perspective for Z-axis animation -->
    <div ref={containerRef} style={{ perspective: '1000px' }} className="relative z-10 pt-8 pb-16">
      <div className="grid grid-cols-2 gap-5 md:grid-cols-3 lg:grid-cols-5 mx-auto max-w-[1400px] px-5">
        {MOCK_CONTACTS.map(contact => (
          <div key={contact.id} className="contact-card">
            <ContactCard
              name={contact.name}
              photo={contact.photo}
              starLevel={contact.starLevel}
              starColor={contact.starColor}
            />
          </div>
        ))}
      </div>
    </div>
  </div>
  ```

**3. GSAP ScrollTrigger entrance animation in `useGSAP`:**
- Scope to `containerRef`
- `gsap.set('.contact-card', { opacity: 0, z: -100, scale: 0.9 })` -- initial state (pushed back in Z-space)
- `ScrollTrigger.batch('.contact-card', { ... })`:
  - `interval: 0.1` (time window for batching)
  - `onEnter: (batch) => gsap.to(batch, { opacity: 1, z: 0, scale: 1, duration: 0.6, stagger: { each: 0.08, grid: 'auto', from: 'start' }, ease: 'back.out(1.2)', overwrite: true })`
  - `once: true` (only animate entrance once per card)

**4. Aurora theme special handling:**
- For the Aurora theme, the page background needs the warm gradient. Add conditional background handling: if `data-theme="aurora"`, the page background should use `var(--gradient-grid-bg)` instead of `var(--color-grid-bg)`. Achieve this by setting the page wrapper background to `var(--color-grid-bg)` AND adding a CSS rule in globals.css: `[data-theme="aurora"] .grid-page-bg { background: var(--gradient-grid-bg) !important; }`. Add class `grid-page-bg` to the page wrapper div.

**Pitfall avoidance:**
- Cards start with `opacity: 0` via CSS class (prevents flash before GSAP runs -- see Pitfall 6 in RESEARCH.md)
- `perspective: 1000px` on parent container (Pitfall 1 -- without it, `z` transform has no visual effect)
- GSAP plugin registration at module scope in `'use client'` file (Pitfall 2)
- `useGSAP` with `scope: containerRef` handles cleanup automatically (Pitfall 3)
