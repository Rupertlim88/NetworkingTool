**1. Create `lib/theme.ts`** — Theme type definitions and constants:
```typescript
export type Theme = 'teal' | 'midnight' | 'aurora';

export const THEMES: readonly Theme[] = ['teal', 'midnight', 'aurora'] as const;

export const THEME_LABELS: Record<Theme, string> = {
  teal: 'Teal',
  midnight: 'Midnight',
  aurora: 'Aurora',
};

export const DEFAULT_THEME: Theme = 'teal';

export const THEME_STORAGE_KEY = 'theme';
```

**2. Create `components/ThemeProvider.tsx`** — Client component:
- `'use client'` directive as first line
- Creates a `ThemeContext` with `{ theme: Theme; setTheme: (t: Theme) => void }`
- `ThemeProvider` component wraps children in context provider
- On mount (`useEffect`), reads `localStorage.getItem(THEME_STORAGE_KEY)`, validates against `THEMES` array, calls `setTheme` if valid
- `setTheme` handler: updates React state, calls `document.documentElement.setAttribute('data-theme', theme)`, calls `localStorage.setItem(THEME_STORAGE_KEY, theme)`
- Exports `useTheme` hook via `useContext(ThemeContext)`
- Default state: `'teal'` (but the flash prevention script handles initial render — see below)
- Import `Theme`, `THEMES`, `DEFAULT_THEME`, `THEME_STORAGE_KEY` from `lib/theme.ts`

**3. Modify `app/layout.tsx`**:
- Import `ThemeProvider` from `@/components/ThemeProvider`
- Import `Script` from `next/script`
- Add a `<Script>` tag with `strategy="beforeInteractive"` inside `<head>` that runs BEFORE paint:
  ```javascript
  (function() {
    try {
      var t = localStorage.getItem('theme');
      if (t && ['teal','midnight','aurora'].indexOf(t) !== -1) {
        document.documentElement.setAttribute('data-theme', t);
      }
    } catch(e) {}
  })()
  ```
  This prevents flash of default theme by setting `data-theme` before React hydration.

  IMPORTANT: Next.js App Router uses `next/script` — the `beforeInteractive` strategy only works inside a custom `_document` or layout `<head>`. If `beforeInteractive` does not work in App Router (it may not for inline scripts), use a raw `<script dangerouslySetInnerHTML>` tag inside the `<head>` element of the html. The key requirement is: the script MUST run before the browser paints.

- Wrap `{children}` with `<ThemeProvider>{children}</ThemeProvider>` inside the `<body>` tag
- Keep `RootLayout` as a server component (NOT 'use client') — ThemeProvider is a client component boundary
- Keep existing Geist font configuration and metadata unchanged

**Important constraints:**
- Do NOT add 'use client' to layout.tsx — it must remain a server component
- ThemeProvider must handle the case where localStorage is unavailable (SSR, incognito) gracefully
- The flash prevention script must be framework-agnostic (vanilla JS, no React, no imports) since it runs before hydration
